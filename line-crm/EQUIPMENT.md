# 100台云手机 — 服务器配置方案

## 一、资源需求计算

### 单台服务器负载

每台服务器运行 1 个 Docker 实例，管理 15 台云手机：

| 资源 | 消耗 | 说明 |
|------|------|------|
| CPU | ~0.3 核/台手机 | ADB 连接 + uiautomator2 + Flask 请求 |
| 内存 | ~150MB/台手机 | uiautomator2 连接缓存 + Python 进程 |
| 网络 | 极低 | ADB 只传文本指令和 XML dump，不传图片 |
| 磁盘 | 20GB | 日志 + 名单 + Docker 镜像 |

### 100 台总需求

| 项目 | 计算 | 结果 |
|------|------|------|
| 服务器数量 | 100 ÷ 15 ≈ 7 台 | **推荐 7 台（每台管 15 台手机）** |
| 每台 CPU | 15 × 0.3 = 4.5 核 | **8 核够用** |
| 每台内存 | 15 × 200MB + 2GB 系统 ≈ 5GB | **8GB 够用** |
| 备用服务器 | 防止单点故障 | **+1 台备用** |

## 二、推荐配置

### 方案 A：云服务器（推荐）

直接买 VPS，不需要自己买硬件。

| 配置项 | 规格 |
|--------|------|
| CPU | 8 核 |
| 内存 | 8 GB |
| 硬盘 | 50 GB SSD |
| 带宽 | 5 Mbps（不限流量） |
| 系统 | Ubuntu 22.04 |
| 数量 | **7 台主力 + 1 台备用 = 8 台** |
| 参考价格 | $25-40/月/台（日本/香港机房） |

推荐机房：日本东京（离台湾近，ADB 延迟低）或香港。

### 方案 B：本地物理机

如果不想月付，一次性买断。

| 配置项 | 规格 |
|--------|------|
| CPU | i5-12400 / R5 5600（6 核 12 线程） |
| 内存 | 16 GB DDR4 |
| 硬盘 | 256 GB NVMe SSD |
| 网络 | 100M 宽带 |
| 系统 | Ubuntu 22.04 |
| 数量 | **3 台**（每台管 30-35 台手机） |
| 参考价格 | ¥2500-3500/台 |

本地机 CPU 更强，一台可管 30 台。但需要稳定的网络和电源。

## 四、别忘了云手机成本（这才是大头）

| 项目 | 单价 | 100 台/月 |
|------|------|----------|
| 云手机 | ¥15-30/台/月 | ¥1,500-3,000 |
| 台湾住宅 IP | ¥30-60/个/月 | ¥3,000-6,000 |
| **手机+IP 合计** | | **¥4,500-9,000/月** |

服务器是零头，云手机和 IP 才是持续开销。

## 五、部署方案（推荐）

```
┌──────────────────────────────────────┐
│          一台管理终端（你现在的电脑）     │
│    SSH 连入各服务器，统一管理部署        │
└──────────────────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    ▼               ▼               ▼
服务器 A         服务器 B         服务器 C
(管 15台)       (管 15台)       (管 15台)
Docker 1        Docker 2        Docker 3
    │               │               │
 15台云手机      15台云手机      15台云手机
    ...（共 7 台服务器 × 15 = 105 台）...

每台服务器完全相同配置，部署同一套代码，独立运行。
坏一台只影响 15 台，随时可切换。
```

## 七、给你的建议

1. **先别买 8 台**：先用你现在这台扩到 20-30 台试跑，看瓶颈在哪
2. **云手机服务商谈批发价**：100 台的量可以砍到 ¥10/台/月以下
3. **IP 也可以谈**：100 个住宅 IP 有议价空间
4. **代码不需要急着改**：当前架构管 30 台没问题，到 50 台以上再考虑拆分
