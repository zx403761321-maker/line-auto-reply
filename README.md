# LINE 自动获客系统 — 白话版说明书

---

## 这是什么？

一个自动帮你操作 LINE 的机器人。它能在 5 台手机上自动加好友、自动聊天回复、发现意向客户就通知你。

**相当于：雇了 5 个虚拟员工，每人每天加 20 个好友，有人发消息自动回，碰到想贷款的马上告诉你。**

---

## 怎么运作的？（用送外卖来比喻）

把整个系统想象成一个外卖店：

| 外卖店 | 你的系统 |
|--------|---------|
| 厨房（做菜的地方） | 云服务器 — 所有指令从这里发出 |
| 5 个外卖小哥 | 5 台云手机 — 负责"跑腿"加好友 |
| 外卖小哥的电瓶车 | ADB 连接 — 服务器遥控手机的通道 |
| 外卖小哥的手机卡 | 台湾住宅 IP — 让 LINE 以为是台湾真人在用 |
| 出餐流程 | cron + loop — 定时自动执行任务 |
| 菜品配方 | DeepSeek AI — 自动帮你想怎么回复 |

---

## 为什么这样设计？

### 1. 为什么用云手机而不是真手机？

真手机要买设备、插电、怕被封。云手机在云端，封号了 3 分钟重建一个，成本更低。而且自带台湾 IP。

### 2. 为什么用台湾住宅 IP？

LINE 会检测你从哪上网。用台湾住宅 IP，LINE 以为你是台湾普通用户，不会封你。

### 3. 为什么要 5 台手机？

1 台手机加太多好友会被 LINE 怀疑。分散到 5 台，每台每天只加 20 个，安全。而且不同人设（美玲、雅琳...）让客户感觉跟真人聊天。

### 4. 为什么服务器在日本？

日本离台湾近，网络延迟低。而且 LINE 不检测你服务器在哪，只检测手机 IP。

### 5. 为什么用 Docker？

Docker 就像一个集装箱——把所有需要的软件打包在一起。换服务器的时候，直接搬过去就行，不用重新安装配置。

### 6. 为什么要装 ADB？

ADB（Android Debug Bridge）= 遥控器。服务器通过它远程操作手机：点击、滑动、打字。没有它，服务器碰不到手机。

### 7. 为什么要分时段添加？

5 台手机如果同时在加好友，万一有问题你全挂了。分到 9:00、11:00、13:00、15:00、17:00 各跑一台，互不干扰。

### 8. 为什么回复要在添加完之后？

添加好友的时候，手机会被占用（一直点来点去）。如果同时回复消息，可能会"打架"——两个操作抢一个手机，结果全乱套。

### 9. 为什么要识别绿数字而不是红点？

LINE 的消息提示：底部聊天 Tab 上的**红色数字**=总共有几条未读。聊天列表里每条消息右边的**绿色数字**=这个人发了几条。

**红数字告诉你有新消息，绿数字告诉你找谁。**

只回复绿数字 < 10 的人（超过 10 条的很可能是广告群）。

### 10. 为什么要过滤 < 10 的绿数字？

你新加的好友不可能一上来发 100 条消息。数字大的（100+）基本都是系统通知、官方号广告、群聊。过滤掉省得回复机器人。

### 11. 为什么用 DeepSeek 而不是 ChatGPT？

DeepSeek 便宜，中文好，支持繁体。而且 API 可以直接调。

### 12. 为什么不做成通知推送？

你先看看有没有意向客户。量大了再加钉钉推送。现在线索存在文件里，打开就能看。

### 13. 为什么不用 OpenClaw 了？

装的时候以为需要它做调度。后来发现 cron + Python 脚本更简单直接。就停了，省内存。

## 能做什么？

### 功能 1：自动加好友

每天指定时间，自动在 LINE 上搜索 ID → 加好友 → 发打招呼。

- 不存在的 ID 自动跳过
- 页面跑偏了自动重启重来
- 每台手机自动改名（把好友名改成 LINE ID）
- 每台独立进度，不会重复加同一个人

### 功能 2：自动回复

有人发消息 → 自动检测 → 读内容 → DeepSeek 生成回复 → 发送。

- 发表情 → 回"你好"
- 文字聊天 → AI 自动对话
- 说想贷款 → 正常聊天 + 记录线索

### 功能 3：意向客户记录

AI 自动判断对方有没有贷款意向（L1 到 L5）。

- L1 = 完全无关
- L2 = 普通聊天
- L3 = 有兴趣 → 记录为线索
- L4 = 很想贷款 → 记录为线索
- L5 = 马上要办 → 记录为线索

L3 以上自动存到文件，你打开就能看到谁想贷款。

```
/root/line-crm/                    ← 所有脚本和配置
  ├── scripts/
  │   ├── daily_add.sh             ← cloud-01 添加脚本
  │   ├── daily_add_cloud-02.sh    ← cloud-02 添加脚本
  │   └── ...                      ← 03/04/05 同理

  │   ├── state/                   ← 进度记录（加到哪里了）

/workspace/openclaw/               ← Docker 项目
  └── services/adb-bridge/
      ├── main.py                  ← 核心大脑（所有功能代码）
      └── loop.py                  ← 自动回复循环
```

## 新手看这里：怎么操作

### 日常你什么都不用做

系统全自动：每天早上 9:00 开始加好友，11:00 开始自动回复，23:00 睡觉。

### 偶尔你要做的

**看加了多少人：**
```bash
tail /root/line-crm/logs/daily_add.log
```

**看有没有意向客户：**
```bash
cat /root/line-crm/data/leads/leads.jsonl
```

**系统崩了重启：**
```bash
cd /workspace/openclaw && docker compose restart adb-bridge
```

**加新的 LINE ID 到名单：**
编辑 `/root/targets.txt`，一行一个 ID，加进去就行。

**加一台新手机：**
```bash
adb connect 新IP:端口
curl -X POST http://127.0.0.1:8899/device/register \
  -H "Content-Type: application/json" \
  -d '{"device_id":"cloud-06","addr":"IP:端口","type":"cloud","label":"新手机"}'
```
然后复制一份 daily_add 脚本，改 DEVICE 名字就行。

## 常见问题（白话版）

**Q: 为什么有时候加好友加到社群里去了？**
A: 页面加载慢了，点错了地方。程序会自动检测，发现错了就重启重来。

**Q: 为什么有的好友没改成名？**
A: 笔形图标位置会跟着名字长短变化，有时候没点到。不影响加好友，LINE ID 在日志里能查到。

**Q: 为什么要 5 分钟才检查一次消息？**
A: 太频繁手机会被占用太多时间，而且消息一般不需要秒回。

**Q: 换服务器怎么搬？**
A: 打包 `/workspace/openclaw` 和 `/root/line-crm` 两个文件夹，新服务器解压，装 Docker 和 ADB，启动就行。
